# সুইফট ভ্যারিয়েবল কনস্টান্টস এবং লিটেরালস

## ১ সুইফট ভ্যারিয়েবলস

প্রোগ্রামিং এর ভাষায় ভ্যারিয়েবল হচ্ছে কোন ডাটা রাখার জন্য একটি নির্দিষ্ট মেমোরি এরিয়া।\
উদাহরণঃ

```swift
var num = 19
```

এখানে, `num` একটি ভ্যারিয়েবল যা `19` ভ্যালুটিকে স্টোর করে নির্দিষ্ট মেমোরি এরিয়াতে।

### কিভারে সুইফটে ভ্যারিয়েবল লেখা হয়?

`var` কীওয়ার্ড দিয়ে সুইফটে একটি ভ্যারিয়েবল ডিফাইন করা হয়ে থাকে, উদাহরণ,

```swift
var name: String
var id: Int
```

এখানে,

- `name` ভ্যারিয়েবলটি একটি স্ট্রিং টাইপের ভ্যারিয়েবল, যা টেক্সট ডাটার জন্য ব্যবহৃত হয়।
- `id` একটি ইন্টিজার টাইপের ভ্যারিয়েবল, যা ইন্ট ডাটার জন্য ব্যবহৃত হয়।
  ভ্যারিয়েবল ডিফাইন করার পর এটি কি ডাটা টাইপের হবে তা ডিফাইন করে দেয়া যায়। পরবর্তীতে ভ্যারিয়েবল সম্পর্কে বিষদ আলোচনা করা হয়েছে।

:::tip

**নোট:** ভ্যারিয়েবল একবার ডিফাইন করলে এটির ডাটা টাইপ পরবর্তীতে পরিবর্তন করা যাব না।

:::

### ভ্যারিয়েবলে ভালু এসাইন

`=` অপারেটর ব্যবহার করে কোন ভ্যারিয়েবলে ভালু এসাইন করা হয়েছে।

```swift
var name = "Ferdous"
```

উপরের উদাহরণে খেয়াল করলে দেখবেন আমার ভ্যারিয়েবল ডিফাইন করার সময় এটি কোন ডাটা টাইপের হবে সেটি উল্লেখ করিনি।
এখানে সুইফট কম্পাইলার অটোমেটিক ভাবে এসাইন করে দিচ্ছে।

### ভ্যারিয়েবলের ভালু পরিবর্তন

খুব সহজেই কোন ভ্যারিয়েবলের ভালু পরিবর্তন করা যায় একই টাইপের অন্য কোন ভালু এসাইন করার মাধ্যমে।
যেমনঃ

```swift
var name = "Ferdous"
name = "Akash"
print(name)
```

আউটপুট

```bash
Akash
```

এখানে আমরা `name` ভ্যারিয়েবলের আগের ভালু `Ferdous` পরিবর্তন করে নতুন ভালু `Akash` এসাইন করেছি।

### ভ্যারিয়েবল নামকরণের নিয়ম

ভ্যারিয়েবল নামকরণের ক্ষেত্রে কিছু নিয়ম মানতে হয়। কারণ আমরা ইচ্ছা মতো ভ্যারিয়েবলের নাম দিতে পারিনা।

**১** ভ্যারিয়েবল অবশ্যই কোন অক্ষর, আন্ডারস্কোর অথবা ডলার সিম্বল দিয়ে শুরু হতে হবে। এছাড়া অন্য কোন সিম্বল দিয়ে ভ্যারিয়েবল শুরু হতে পারবেনা। যেমনঃ

```swift
// valid
var a = "hello"
var _a = "hello"
var $a = "hello"

// invalid
var @name = "Ferdous"
```

**২** ভ্যারিয়েবলের নাম কখনো সংখ্যা দিয়ে শুরু হতে পারবেনা। যেমনঃ

```swift
// invalid
var 1a = "hello" // throws error

// valid
var a1 = "hello"
```

**৩** সুইফট একটি কেস সেনসেটিভ ল্যাঙ্গুয়েজ, এখানে `a` এবং `A` দ্বারা দুইটি পৃথক ভ্যারিয়েবলকে বোঝানো হয়ে থাকে। যেমনঃ

```swift
var A = 19
var a = 9
print(A) // 19
print(a) // 9
```

**৪** সুইফটের রিসার্ভ কীওয়ার্ড গুলোকে আমরা ভ্যারিয়েবলের নাম হিসাবে ব্যবহার করতে পারবোনা। যেমনঃ

```swift
// invalid
var String = "Ferdous Mahmud Akash"
var public = True

// valid
var myString = "Ferdous Mahmud Akash"
var isPublic = True
```

:::tip

- ভ্যারিয়েবলের নাম সবসময় অর্থবহ হওয়া উচিত, যেন ভ্যারিয়েবলে কী ভালু এসাইন করেছি সেটি ভ্যারিয়েবলের নাম থেকেই অনুমান করা যায়। যেমনঃ `passwordLenght`, `isValidPassword`, `currentLocation` ইত্যাদি।
- সুইফটে আমরা [ক্যামেল কেস](https://en.wikipedia.org/wiki/Camel_case) নেমিং কনভেনশন ব্যবহার করে থাকি। এজন্য আমরা যখন ভ্যারিয়েবল ডিফাইন করবো তখন ক্যামেল কেস ব্যাবহার করবো। এটি আমারদের কোডের কন্সিস্টেন্সি বজায় রাখে, যখন টিম এর কোডে কান্ট্রিবিউট করা হয়। উদাহরণঃ

```swift
// Camel case
let camelCaseVariable = 10

// Snake case
let snake_case_variable = 20

// Pascal case
let PascalCaseVariable = 30

// Kebab case (less common in Swift)
let kebab_case_variable = 40
```

:::

## ২ সুইফট কনস্টান্টস

### কিভারে সুইফটে কনস্টান্টস লেখা হয়?

কনস্টান্টস একটি স্পেশাল টাইপের ভ্যারিয়েবল, যার ভালু একবার এসাইন করার পর পরবর্তীতে পরিবর্তন করা যায়না। যেমনঃ

```swift
let num = 19

num = 10 // Compile time error
```

আউটপুটঃ

```bash
/dev/main.swift:3:1: error: cannot assign to value: 'num' is a 'let' constant
```

ভ্যারিয়েবলের মতো আমার কনস্টান্টস কী ডাটা টাইপের হবে সেটি ডিফাইন করে দিতে পারি কিন্ত, কনস্টান্টসে ভালু এসাইন করার আগে এটিকে ব্যবহার করতে পারিনা। যেমনঃ

```swift
let bookName: String
print(bookName) // error: constant 'bookName' used before being initialized

let bookName: String = "সুইফট বাংলা"
print(bookName) // output: সুইফট বাংলা

```

:::tip

- যদি কোন প্রোগ্রামে একটি ভ্যারিয়েবলের ভালু পরিবর্তন করার দরকার না পড়ে, তাহলে এটিকে `let` কনস্টান্টস হিসাবে ডিফাইন করা ভালো।
- ভ্যারিয়েবল নামকরণের নিয়মগুলো কনস্টান্টস এর ক্ষেত্রেও প্রযোজ্য।

:::

## ৩ সুইফট লিটারাল

কোন ভ্যারিয়েবলে যে ভ্যালু এসাইন করা হয় তাকে আমরা লিটারাল বলতে পারি। এটি ক্যারেক্টার, স্ট্রিং, নাম্বার ইত্যাদি হতে পরে।
কোন প্রোগ্রামের যে কোনো একটি নিদৃষ্ট ভ্যালুকে লিটারাল বলা হয়। উদাহরণ:

```swift
let name = "Ferdous"
```

উপরের উদাহরনে `name` একটি ভ্যারিয়েবল এবং `Ferdous` একটি লিটারেল।
এখন আমরা বিভিন্ন প্রকার লিটারাল সম্পর্কে জানার চেষ্টা করবো।

### ইন্টিজার লিটারাল

যেসব সংখ্যা লিটারালে ডেসিমাল(বেস ১০), অক্টাল(বেস ৮), হেক্সাডেসিমেল(বেস ১৬), বাইনারি(বেস ২) থাকে তাদেরকে ইন্টিজার লিটারাল বলে।
সুইফটে `৪` ধরনের ইন্টিজার লিটারাল ব্যবহার করা হয়।

| টাইপ          |   উদহারণ    |    মন্তব্য     |
| :------------ | :---------: | :------------: |
| ডেসিমাল       | 5, 10, -68  |  পূর্ণ সংখ্যা  |
| বাইনারি       | 0b101, 0b11 |   শুরু `0b`    |
| অক্টাল        |    0o13     | শুরু with `0o` |
| হেক্সাডেসিমেল |    0x13     | শুরু with `0x` |

### ফ্লোটিং পয়েন্ট লিটারাল

যেসব সংখ্যা লিটারাল দশমিক সংখ্যা নিয়ে কাজ করে তাদেরকে ফ্লোটিং পয়েন্ট লিটারাল বলে। যেমন:

```swift
let valueOfPi: Float = 3.14
```

এখানে `3.14` একটি ফ্লোটিং পয়েন্ট লিটারাল, যা `valueOfPi` কনস্টান্টসে অ্যাসাইন করা হয়েছে।

### বুলিয়ান লিটারাল

সুইফটে দুই ধরনের বুলিয়ান লিটারাল হয়ে থাকে, `true` এবং `false`। যেমন:

```swift
var isPassed: Bool = true
```

এখানে `true` একটি বুলিয়ান লিটারাল, যা `isPassed` ভ্যারিয়েবলে অ্যাসাইন করা হয়েছে।

### স্ট্রিং এবং ক্যারেক্টার লিটারাল

সকল ইউনিকোড ক্যারেক্টারকে ক্যারেক্টার লিটারাল বলা হয়, এবং এটি বাবহারের ক্ষেত্রে `""` ডাবল কোট চিহ্ন ব্যবহৃত করা হয়।
যেমন:

```swift
var firstLetter: Character = "A"
```

এখানে `A` একটি ক্যারেক্টার লিটারাল যা `firstLetter` ভ্যারিয়েবলে অ্যাসাইন করা হয়েছে।

একইভাবে এক বা একাধিক ক্যারেক্টারের সমষ্টিকে যদি ডাবল কোট দিয়ে সমষ্টিবদ্ধ করা হয় তখন তাকে স্ট্রিং লিটারাল বলে।
যেমন:

```swift
var myString = "আমি বাংলায় সুইফট শিখছি"
```

এখানে `আমি বাংলায় সুইফট শিখছি` একটি স্ট্রিং লিটারাল, যা `myString` ভ্যারিয়েবলে অ্যাসাইন করা হয়েছে।
